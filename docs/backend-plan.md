# Backend Plan

## Goal

Build a Java Spring Boot backend for KiwiHire Coach so application data, resume reviews, and interview questions can be saved in a database instead of only localStorage.

## Main Objects

- User
- JobApplication
- ResumeReview
- InterviewQuestion

## API Endpoints

- POST /api/auth/register
- POST /api/auth/login
- GET /api/applications
- POST /api/applications
- GET /api/applications/{id}
- POST /api/resume-reviews
- GET /api/resume-reviews
- GET /api/resume-reviews/{id}

## Database Tables


### users

- id
- email
- password_hash
- created_at

### applications

- id
- user_id
- company
- role_title
- status
- job_description
- created_at

### resume_reviews

- id
- user_id
- application_id
- resume_text
- created_at

### interview_questions

- id
- resume_review_id
- question
- reason
- answer_guide
- related_skill

## Object Relationships

- One User can have many JobApplications.
- One User can have many ResumeReviews.
- One JobApplication can have many ResumeReviews.
- One ResumeReview can have many InterviewQuestions.

The user_id field connects users to applications and resume reviews.
The application_id field connects a resume review to a job application.
The resume_review_id field connects interview questions to one resume review.

## First Milestone

The first backend milestone is to build a simple Spring Boot API for job applications.

Scope:
- Create a Spring Boot backend project.
- Create a JobApplication model.
- Create endpoints to create and list job applications.
- Store application data in PostgreSQL.
- Add basic tests for the service and controller layers.

Out of scope for the first milestone:
- JWT authentication
- Kubernetes deployment
- CI/CD
- Resume review generation

## API Status Codes

### Success

- 200 OK: request succeeded
- 201 Created: new resource created

### Client Errors

- 400 Bad Request: request body is invalid
- 401 Unauthorized: user is not logged in
- 404 Not Found: resource does not exist

## Sample Seed Data

The large local seed data has been moved to:

`backend/src/main/resources/data.sql`

Current seed set:

- 50 users
- 90 job applications
- 60 resume reviews
- 120 interview questions

Notes:

- The `id` values are fixed only for local development seed data.
- In the real database, IDs should be generated by the database.
- The frontend should not send `id` or `created_at` when creating a record.
- Passwords are fake placeholders and must be properly hashed in real code.
- The API response can use camelCase, but database columns should stay snake_case.
